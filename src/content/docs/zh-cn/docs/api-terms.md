欢迎使用 **Open Endfield Map**（以下简称“我们”、“本项目”或“OEM”）提供的应用程序编程接口（API）。通过访问、集成或以任何方式使用 OEM API，即表示您（“开发者”或“您”）已阅读、理解并同意接受本《API 使用条款》（以下简称“本条款”）的完全法律约束。如果您不同意本条款的任何内容，请立即停止访问和使用本 API。

## 账户与访问凭证
1. **凭证安全**：如果 OEM 为您分配了 API 密钥（API Key）或访问令牌，您必须严格保密。您不得与任何第三方共享、出售、转让或以其他方式泄露您的 API 凭证。
2. **凭证责任**：您应对使用您的 API 凭证发生的所有活动负全部责任。如果发现凭证泄露或未经授权的使用，您必须立即通知我们。

## 允许的使用
您可以出于个人、学习及非商业目的访问 OEM API。我们鼓励开发者在 OEM 现有数据的基础上，构建能提供实质性创新体验的衍生工具（例如：游戏路线规划器、资源养成计算器、数据分析工具、Discord/QQ 机器人等）。您的应用必须为终端用户提供 OEM 官方平台之外的**附加价值**。

## 禁止的使用
您在使用 OEM API 时，严格禁止以下行为：
1. **任何形式的商业化**：您不得利用 OEM API 或其提供的数据产生任何直接或间接的收入。包括但不限于：在应用中植入广告、设置付费订阅（Paywalls）、强制性赞助/打赏、或将应用作为付费软件出售。如需商业授权，必须事先取得 OEM 团队的书面许可。
2. **镜像、搬运或替代品开发**：您不得使用本 API“原样”复制、展示或重新分发 OEM 的地图内容与交互界面。严禁利用本 API 开发旨在替代、分流 OEM 官方网站流量的套壳复制品。
3. **批量抓取与数据集转售**：您不得大规模下载、抓取、提取本 API 的基础数据（如全量坐标点位、图鉴文本），并将其作为您独立的离线数据集，或通过您自己的 API 进行二次分发。
4. **滥用与破坏**：您不得以任何可能破坏、使之超载或损害 OEM 服务器及网络的方式使用本 API（包括但不限于 DDoS 攻击、利用漏洞注入恶意代码等）。

## 调用限制与缓存策略
1. **遵守频率限制**：您必须遵守 OEM 在 API 响应头（Headers）或官方文档中规定的调用频率限制（Rate Limits）。严禁通过多 IP 轮换、伪造 User-Agent 等手段规避限制。
2. **合理缓存**：为了保护我们的服务器资源，您应当在您的应用中实施合理的本地缓存（Caching）策略。避免对静态或极少变更的数据进行高频重复请求。

## 知识产权与品牌标识
1. **数据所有权**：通过本 API 获取的所有地图基础数据、图片资源、标记信息及社区共创内容（UGC）的所有权和知识产权，均归属于 Open Endfield Map 及其各自的贡献者。
2. **品牌边界**：未经事先书面同意，您不得在您的应用名称、Logo 或域名中擅自使用“Open Endfield Map”或可能引起混淆的相似名称。您不得暗示您的应用是 OEM 的“官方产品”或获得了 OEM 的官方背书。

## 署名要求
如果您使用本 API 构建了公开服务，您必须在应用界面的显著位置（如页脚、关于页面）提供清晰的署名，并包含指向 OEM 官方网站的有效超链接。
*示例声明：“本应用的地形与坐标数据由 [Open Endfield Map](https://openendfieldmap.com) 提供支持。”*

## 隐私与用户数据
如果您的应用涉及处理最终用户的个人信息，您必须提供合法合规的《隐私政策》。您不得利用 OEM API 试图收集、反编译或获取 OEM 平台其他用户的个人隐私数据或账户信息。
